# Code of Conduct

## Our Pledge

We want Memoire to be useful for developers, designers, researchers, and maintainers working on design-system automation. Participation should be respectful, technical, and focused on improving the project.

## Standards

Expected behavior:

- Be respectful and direct.
- Assume good intent while still giving concrete technical feedback.
- Keep issues, discussions, and PRs scoped to actionable project work.
- Disclose conflicts of interest when submitting directory listings, benchmarks, or comparisons.

Unacceptable behavior:

- Harassment, threats, or personal attacks.
- Spam, fake engagement, fake stars, or achievement farming that harms external maintainers.
- Public disclosure of security vulnerabilities before a fix or coordinated disclosure path exists.

## Enforcement

Maintainers may edit, hide, or remove comments and may block participants who repeatedly violate these rules. Security issues should follow `SECURITY.md`.
